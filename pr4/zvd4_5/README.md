# Завдання 4.5: Безпечна робота з realloc

### Опис
Продемонстровано сценарій, коли `realloc` повертає `NULL` через неможливість виділити величезний блок пам'яті.

### Результат
Незважаючи на відмову системи, початковий вказівник (наприклад, `0x5cb83e5b32a0`) залишився валідним і пам'ять за ним не була втрачена. Це підтверджує необхідність використання тимчасової змінної при виклику `realloc`.
