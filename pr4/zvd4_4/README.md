# Завдання 4.4: Проблема звільнення у циклах

### Опис
Проаналізовано код, де `free(ptr)` викликається без подальшого занулення вказівника.

### Висновок
Після виклику `free()`, вказівник `ptr` продовжує зберігати стару адресу (висячий вказівник). Умова `if (!ptr)` у наступній ітерації циклу не спрацює, що призведе до помилок доступу до пам'яті.
**Рішення**: Після `free(ptr)` завжди додавати `ptr = NULL;`.
