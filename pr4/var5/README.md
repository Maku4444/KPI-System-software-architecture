# Варіант: Сторожові сторінки (mmap + mprotect)

### Опис
Продемонстровано механізм захисту пам'яті від переповнення буфера за допомогою сторожових сторінок (guard pages).

### Як це працює
1. Виділено три сторінки пам'яті через `mmap()`.
2. За допомогою `mprotect()` крайнім сторінкам встановлено права `PROT_NONE`.
3. Робоча область знаходиться посередині.
4. Будь-яка спроба звернення до адрес за межами робочої сторінки призводить до негайного системного переривання `SIGSEGV`.

### Результат виконання
Програма успішно записує дані в дозволену область, але миттєво падає з помилкою `Segmentation fault` при спробі доступу до сторожової сторінки.
