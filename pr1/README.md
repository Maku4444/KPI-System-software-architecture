Звіт про виконання практичної роботи №1  
1. Підготовка робочого середовища  
На віртуальну машину (VirtualBox) успішно встановлено операційну систему Ubuntu Linux.
Налаштування системи: Встановлено актуальні версії пакетів та необхідні драйвери для коректної роботи ОС.
Середовище розробки: Встановлено інструментарій для роботи з мовою ANSI C (пакет build-essential, що включає GCC та GNU Make).

Мета завдання була досягнута створенням окремих файлів .с та використанням в деяких з них заголовків .h з подальшим їх лінкуванням через Makefile та відстеженням їх залежностей один від одного при компілюванні.  
Скріншот виконання:
![photo_2026-02-04_17-53-51](https://github.com/user-attachments/assets/90fc667b-88ad-44ab-9e4a-3b9b1abac9ea)  
На скріншоті продемонстровано процес збирання та перевірку механізму відстеження залежностей у терміналі Ubuntu. Весь процес розділено на чотири етапи:  
-Повне збирання проекту (команда make): Оскільки об'єктних файлів ще не існує, утиліта по черзі компілює всі шість модулів (main.c, input.c, output.c, process1.c, process2.c, utility.c).  
-Зміна process1.c: Після оновлення дати зміни файлу process1.c, make визначає, що застарів лише один об'єктний модуль. Система перекомпілює тільки process1.o та заново проводить лінкування проекту.  
-Перевірка залежності process1.h: Команда touch process1.h імітує зміну інтерфейсу. Система автоматично перекомпілює три залежні модулі: main.o, process1.o та utility.o.  
-Перевірка залежності list.h: Остання перевірка показує, що зміна list.h ініціює перекомпіляцію лише тих модулів, що використовують цей список (input.o та process2.o), не чіпаючи решту проекту.
