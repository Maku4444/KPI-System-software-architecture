# Завдання 3.2: Профілювання лімітів ресурсів за допомогою perf

### Опис експерименту
У межах завдання було досліджено вплив ліміту віртуальної пам'яті (`RLIMIT_AS`) на виконання процесів у Docker-контейнері за допомогою утиліти `perf`.

### Результати тестування
1. **Критичне обмеження (10 МБ)**:
   - Команда: `ulimit -v 10000`
   - Результат: `Segmentation fault`
   - Висновок: Системі не вистачило пам'яті для відображення сегментів коду та даних процесу `perf` і `ls` у віртуальний простір.

2. **Достатній ліміт (200 МБ)**:
   - Команда: `ulimit -v 200000`
   - Результат: Успішне виконання.
   - Статистика `perf`:
     - **Page-faults**: 88 (процес успішно виділив необхідні сторінки).
     - **Task-clock**: 1.01 msec (час роботи процесора).
     - **Cycles**: <not supported> (обмеження віртуалізації VirtualBox для доступу до лічильників циклів).

### Загальний висновок
Експеримент підтвердив, що встановлення лімітів через `ulimit` жорстко контролює доступ до системних ресурсів. Перевищення ліміту призводить до негайного переривання роботи програми операційною системою.
