Завдання 3.1: Дослідження лімітів відкритих файлів у Docker
Мета роботи

Дослідити механізм обмеження системних ресурсів (ulimit) у середовищі Docker-контейнера та проаналізувати поведінку «м'яких» (Soft) і «жорстких» (Hard) лімітів.
1. Початковий аналіз

При старті системи було зафіксовано наступні значення:

    Поточний ліміт (ulimit -n): 1,048,576.

    Soft Limit: 1,048,576.

    Hard Limit: 1,048,576.

2. Маніпуляції з лімітами

В ході роботи було виконано спроби змінити кількість дозволених відкритих файлів:

    Зниження до 3000: Команда ulimit -n 3000 спрацювала успішно, встановивши нову межу для поточної сесії.

    Спроба підвищення до 3001: Отримано помилку Operation not permitted. Це сталося тому, що при зміні ліміту без уточнення типу, система знизила і Hard Limit. Повернення до вищого значення стало неможливим.

    Подальше зниження до 2000: Спрацювало успішно, оскільки це значення менше за поточний Hard Limit (3000).

    Спроба повернення до 3000: Отримано відмову, оскільки нова жорстка межа тепер зафіксована на позначці 2000.

Висновки

    Soft Limit визначає фактичне обмеження для процесу, але його можна підвищувати до рівня Hard Limit.

    Hard Limit виступає абсолютною стелею.
